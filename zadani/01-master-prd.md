# BuildSphere — Master Product Requirements Document

## 1. Produktová definice

BuildSphere je digitální platforma kombinující:

- B2C guided journey,
- profesní marketplace,
- profesní sociální síť,
- vizuální portfolio systém,
- pracovní marketplace,
- team/subcontractor marketplace,
- messaging,
- notifikace,
- projektové workspace,
- reputaci, recenze a verifikace,
- produktizované služby,
- B2B sourcing,
- supplier ecosystem.

## 2. Primární problém

Běžný klient často neví:

- koho hledá,
- jak správně popsat problém,
- jaké podklady potřebuje,
- jaké profese budou potřeba,
- v jakém pořadí postupovat,
- zda je nabídka relevantní,
- zda je profesionál důvěryhodný.

Profesionál často neví:

- kde získat kvalitní leady,
- jak odlišit svou odbornost,
- jak prezentovat reference,
- jak ukázat dostupnost,
- jak najít spolupracovníka,
- jak řídit více typů poptávek.

## 3. Hlavní cíle

### B2C
- snížit nejistotu,
- pomoci definovat potřebu,
- vytvořit kvalitní brief,
- doporučit další krok,
- propojit s vhodnými odborníky.

### B2B / Professional
- budovat reputaci,
- prezentovat portfolio,
- získávat relevantní poptávky,
- hledat práci a kapacity,
- vytvářet projektové týmy.

### Platforma
- vytvářet opakovanou hodnotu,
- budovat síťový efekt,
- monetizovat subscriptions, leady, transakce a B2B služby.

## 4. Anti-goals

Platforma nemá být:

- prostý katalog firem,
- anonymní nástěnka,
- jen job board,
- jen sociální síť,
- jen marketplace řemeslníků,
- automatický stavební poradce vydávající závazná stanoviska,
- generátor falešných cenových odhadů.

## 6. Core product loops

### Loop A: Guide → Brief → Match → Conversation → Zakázka
1. Klient popíše problém.
2. Guide doplní kontext.
3. Vznikne brief.
4. Systém doporučí profese a profesionály.
5. Klient osloví vybrané.
6. Proběhne komunikace.
7. Vznikne spolupráce.

### Loop B: Portfolio → Inspirace → Moodboard → Brief
1. Klient prohlíží projekt.
2. Uloží inspiraci.
3. Vytvoří moodboard.
4. Zvolí „chci něco podobného“.
5. Vznikne předvyplněný guide.

### Loop C: Zakázka → Realizace → Reference → Nový lead
1. Projekt je dokončen.
2. Klient hodnotí.
3. Profesionál publikuje case study.
4. Case study přivede další klienty.
